# Draft Intelligence

A draft-model review workspace for the MLB Draft Model — 2026
projections for 10,734 D1 players, calibrated MLB probabilities, projected
round bands, historical comps, class retrospectives, and full model
transparency cards.

Built as a static Next.js 15 export. No backend: the data bundle is generated
by the Python pipeline and shipped as JSON.

## Views

| Route | What it answers |
|---|---|
| `/board` | Who matters in the 2026 class, under my filters? (virtualized 10.7k-row board, grade tiers, round bands, calibrated MLB%) |
| `/value` | Where will the market misprice outcomes? (projected pick × calibrated MLB%, brush-to-filter) |
| `/classes/{year}` | How did past classes actually turn out? (2021–2026, real outcomes) |
| `/models/{artifact}` | Why should I believe any of this? (backtests, reliability diagrams, importances, use/don't-use) |
| `/methodology` | Glossary + known-limitations register |
| `/player?id=…` | Full dossier: verdict row, percentile bars, seasons, comps, caveats, local notes |

## Honesty rules baked into the UI

- Projected picks always render as **round bands** (± backtest MAE), never slots.
- Every probability is **Platt-calibrated**; the raw number is shown as context, never headline.
- The dashed tick on probability bars marks how often players with that raw
  score *actually* reached MLB.
- Grades are percentile **tiers** (top 1% / 4% / 15% of qualified composites) — there is no "#1 overall."
- Null renders as "—", never 0.

## Development

```bash
npm install

npm run build        # static export -> out/
npm start            # serve the export locally
```

## Data pipeline

The bundle in `public/data/` is generated by
`MLB_Draft_Model/scripts/export_frontend_data.py` (in the modeling repo):

```bash
python3 scripts/export_frontend_data.py   # writes web/public/data/
```

It consumes `projections_2026.json`, the fg/tier2 training sets, and the model
artifacts (backtest, calibration, feature importances), and emits:

- `players_index.json` — slim board/scatter records (~5 MB)
- `players/shard-XX.json` — 64 FNV-1a shards of full dossiers
- `models_manifest.json` — everything the model cards render
- `classes/{year}.json` — drafted classes with outcomes
- `meta.json` — vintage, counts, band widths

To point the site at a remote bundle (e.g. an R2 public bucket) instead of the
bundled copy, set `NEXT_PUBLIC_DATA_BASE` (see `.env.example`).

## Deploy

Static output (`out/`) deploys anywhere. Vercel: framework preset Next.js,
no env needed.
